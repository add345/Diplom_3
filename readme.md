### Автотесты для проверки интерфейса заказа бургера в Stellar Burgers

### Реализованные сценарии

Созданы тесты через Selenium, позволяющие проверить:

- переход по клику на «Конструктор»;
- переход по клику на раздел «Лента заказов»;
- если кликнуть на ингредиент, появится всплывающее окно с деталями;
- всплывающее окно закрывается кликом по крестику;
- при добавлении ингредиента в заказ счётчик этого ингредиента увеличивается.
- при создании нового заказа счётчик «Выполнено за всё время» увеличивается;
- при создании нового заказа счётчик «Выполнено за сегодня» увеличивается;
- после оформления заказа его номер появляется в разделе «В работе».

### Структура проекта

- `tests` - пакет, содержащий тесты, разделенные по стрницам
- `pages` - пакет, содержащий классы для взаиомодействия со страницами
- `locators` - пакет, содержащий локаторы
- `allure_results` - файлы, сгенерированные Allure после прогона тестов

### Запуск автотестов

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск автотестов и создание Allure-отчетов**

>  `$ pytest --alluredir=allure_results`

**Генерация HTML-страницы из Allure-отчетов**

>  `$ allure serve allure_results`

**Примечание**

В документации на API не описано удаление заказа, поэтому после завершения автотестов, созданные заказы не удаляются!
